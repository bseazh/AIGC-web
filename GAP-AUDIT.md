# 芭乐AIGC 产品差距审计

> 审计日期：2026-07-20。对照目标为 `https://yinghai.xin/` 的公开页面、公开浏览器接口和可观察交互；本项目为 `https://aigc.bigapple.store/`。本文只比较产品结构与设计语言，不授权复制对方品牌、OSS 媒体、案例内容、用户数据或服务端提示词。

> 2026-07-20 补充：用户提供原站授权账号后，已实测登录后案例门户、AI 生图/视频/工具目录、任务中心、资产库和模特穿搭表单。本轮据此完成并上线芭乐任务中心、任务详情、资产库和工作台真实导航的首版实现。

## 结论

芭乐AIGC 已经完成底层闭环，但尚未形成完整产品体验。当前可真实使用的主流程只有：邮箱验证注册/密码登录、商品素材上传、商品主图四图生成、积分扣除或失败退款、结果写入 COS。首页宣称的其他能力、工作台导航和业务中心多数仍是静态入口或禁用控件。

用户感知上的“很多东西没有”，主要不是首页视觉问题，而是以下四类断层：

1. 首页没有公开案例流，无法展示真实输入、生成结果和能力覆盖。
2. 工作台存在大量空链接、禁用按钮与硬编码状态。
3. 已持久化的任务和资产没有用户可见的列表页。
4. 合规、客服、账户安全和运营后台尚未完成。

## 取证边界

### 已观察

- 原站公开首页、登录弹窗、移动端布局及 9 个公开合规入口。
- 原站公开 `/api/home`：当前返回 20 个案例，图片 10 个、视频 10 个。
- 原站公开案例字段：`title`、`configKey`、媒体地址、缩略图、描述、公开参数、参考素材和创建时间。
- 原站公开案例覆盖 9 类配置键，包括商品主图、模特穿搭、通用生图、主图详情、口播/种草/复刻类视频。
- 原站登录框包含账号密码、短信登录、图形验证码、企业用途确认、知识产权责任确认及协议确认。
- 芭乐线上首页、登录后工作台、商品主图页以及桌面/390px 手机布局。

### 未观察

- 原站登录后私人工作台、用户任务、资产与管理后台。原站没有提供公开测试账号，受认证保护的内容没有通过猜测或绕过方式访问。
- 原站服务端系统提示词、工作流模板、供应商密钥和其他用户数据。

## P0：必须先补的产品闭环

### 1. 任务中心

- 新增 `/tasks`，使用真实 `generation_tasks` 数据，不显示硬编码角标。
- 支持状态筛选、创建时间、工作流、消耗积分、进度、失败原因和结果缩略图。
- 支持进入任务详情、下载结果、可重试任务重试。
- 工作台“最近任务”和“查看全部”接真实 API。
- 用户离开生成页后仍能在任务中心查看异步进度。

### 2. 内容资产库

- 新增 `/assets`，展示输入素材和生成结果。
- 支持图片预览、类型/时间筛选、下载、删除和来源任务跳转。
- 使用短期 COS 签名 URL，并在服务端校验 `owner_id`。
- 工作台“内容资产”和首页资产入口连接真实页面。

### 3. 工作台去空壳化

- 把侧栏 `<a>` 改为真实 `Link`；没有页面的项目明确标记“即将上线”，不要伪装成可点击。
- 搜索框连接任务/资产搜索，或在未实现前移除。
- 通知按钮连接通知面板，或在未实现前移除。
- “新建任务”打开工具选择器，不能保持禁用。
- 移除硬编码任务角标 `1`；角标来自待处理任务数量。
- “充值”第一阶段改为“积分明细”，真实支付关闭时不要显示不可用充值动作。

### 4. 首页真实案例流

- 新增自有 `/api/home` 和案例数据表/运营配置。
- 首页展示已授权或芭乐自行生成的案例，不复制原站 OSS 媒体。
- 每个案例支持输出预览、输入参考图、功能类型、比例、分辨率、模型和公开参数摘要。
- 第一批至少准备 12 个案例，覆盖商品主图、场景图和模特穿搭；视频能力上线后再加入视频。
- 首页入口点击：未登录打开登录并保留目标路由；已登录直接进入对应工具。

### 5. 正式合规页面

- 新增 `/terms`、`/privacy`、`/aigc-service`、`/content-labeling`、`/complaints`、`/violation-rules`、`/minor-protection`、`/third-party-processors`。
- 内容使用芭乐AIGC自身主体、邮箱、供应商和数据处理方式，不复制原站条文。
- 注册时记录协议版本、同意时间、IP 和 User-Agent。
- 增加输入素材授权、生成内容使用责任和 AI 内容标识说明。
- 页脚加入真实备案号后再上线备案链接；未提供备案号前不要虚构。

## P1：补足首版能力

### 图像工作流

1. 场景图：复用商品主图基础设施，增加背景保真与场景参数。
2. 模特穿搭：服装图、可选模特参考、姿态/场景/比例参数。
3. 高清优化：单图上传、放大倍率、降噪/细节策略。
4. 商品详情页：卖点结构、多张输出、统一风格与顺序管理。
5. 主图 + 详情页：组合工作流和部分成功处理。

每个入口必须同时具备页面、报价、任务状态、结果落库、积分结算、失败退款和资产库展示，不能只新增卡片。

### 账户与安全

- 找回密码和邮箱验证码重置密码。
- 登录失败限流、图形验证码触发策略和异常会话退出。
- 账户资料、安全记录、注销流程。
- 企业用途、素材授权与知识产权责任逐项确认。

### 客服与反馈

- 右下角客服按钮打开真实客服面板，显示 `bseazh@163.com`。
- 增加任务编号、问题类型和附件的反馈/申诉表单。
- 后台可查看与处理申诉状态。

## P2：商业化与运营

- 管理后台：用户、任务、资产、积分调整、案例管理、模型配置、供应商健康、审核和申诉。
- 邮件通知：任务完成、失败、账号安全和申诉结果。
- 真实充值前先完成订单、支付回调幂等、退款和对账；未完成前保持管理员测试积分。
- 视频工作流：带货视频、口播、参考视频复刻；需要独立报价、长任务状态、超时和部分结果策略。
- 内容安全：上传前审核、生成后审核、AI 内容显式/隐式标识和审计记录。

## 已发现的具体缺陷

| 位置 | 问题 | 修复 |
| --- | --- | --- |
| 工作台侧栏 | 工作台、创作工具、任务中心、内容资产没有 `href` | 改为真实路由 |
| 工作台角标 | 固定显示 `1`，同时页面显示“暂无任务” | 从 API 读取真实数量 |
| 工具卡 | 4 张卡中 3 张禁用；首页却宣称 8 项能力 | 标记开发状态或完成对应工作流 |
| 工作台操作 | 充值、新建任务禁用 | 改为可执行命令或暂时移除 |
| 搜索/通知 | 有界面无行为 | 实现或移除 |
| 最近任务 | 永远为空，不读取任务表 | 接入任务列表 API |
| 首页卡片 | 无论入口类型都只打开登录框 | 登录后按工具路由导航 |
| 首页内容 | 只有静态 Unsplash 卡片，没有平台案例 | 使用芭乐自有生成案例流 |
| 合规链接 | 服务协议和隐私政策指向 `#` | 建立正式页面 |
| 客服按钮 | 只有图标，无面板或联系方式 | 打开客服/反馈面板 |
| 手机首页 | 存在约 4px 横向溢出 | 修正根容器/跑马灯宽度 |
| 生成结果 | 仅当前页可见，刷新后缺少恢复入口 | 任务详情与资产库持久展示 |

## 推荐实施顺序

### 阶段 A：产品不再“空”（优先）

1. 任务中心及任务详情。
2. 内容资产库。
3. 工作台真实导航、最近任务和动态角标。
4. 首页自有案例流与登录后正确跳转。
5. 客服面板和 8 个合规页面。

### 阶段 B：从 1 个工具扩至 3 个

1. 场景图。已上线。
2. 模特穿搭。
3. 高清优化或详情页，根据模型供应商能力选择一个。

### 阶段 C：运营和商业化

1. 管理后台与人工积分调整。
2. 内容审核、申诉和 AI 标识。
3. 找回密码、账户安全和邮件通知。
4. 视频工作流。
5. 支付和充值。

## 阶段 A 验收标准

- 登录后所有可见导航和按钮都有明确结果，不存在空 `<a>` 或无行为按钮。
- 用户能在任务中心找到刷新前创建的任务，并从任务详情下载结果。
- 用户能在资产库找到输入素材和生成结果，且不能访问其他用户资产。
- 首页案例全部来自芭乐自有或已授权素材，并可进入对应可用工具。
- 页脚所有链接均指向真实页面，协议版本同意有数据库记录。
- 390px、768px 和 1440px 无横向页面溢出、遮挡和文本截断。
- GitHub Actions 自动部署成功，线上健康接口、登录、任务列表和资产列表通过回归检查。
